# LEVI'S Heritage 프로젝트 기획서 (포트폴리오용)

## 1. 프로젝트 개요
- 프로젝트명: LEVI'S Archive Denim
- 유형: 개인
- 기간: 2026.01.28 ~ 2026.02.02
- 역할: 기획 / 콘텐츠 구성 / UI 디자인 / 퍼블리싱(HTML/CSS/JS) / 인터랙션 구현
- 산출물: 메인 + 서브 페이지(heritage/craft/archive) 멀티페이지

## 2. 배경/문제 정의
- 커머셜 UI보다 브랜드 헤리티지와 서사를 강조하는 페이지 필요

## 3. 목표
- 데님/워크웨어의 물성과 시간성을 시각적으로 표현
- 메인에서 요약, 서브 페이지에서 깊이 있는 서사 제공
- 상업적 요소 최소화, 브랜드 문화/아카이브 중심

## 4. 타겟
- 데님 문화/워크웨어에 관심 있는 20~40대
- 브랜드 상징성과 역사성을 중요하게 보는 사용자
- 쇼핑보다 브랜드 가치를 이해하고 싶은 사용자

## 5. 핵심 컨셉/키워드
- Heritage / Archive / Craft
- Indigo / Denim / Stitch
- Story-driven / Editorial layout

## 6. 정보 구조/페이지 구성
- Home: 스토리 요약, 각 섹션 Read More
- Heritage: 1873~현재 타임라인
- Craft: 원단/염색/봉제 공정 상세
- Archive: 룩북/텍스처/문화 아카이브

## 7. 주요 기능/인터랙션
- Hero Reveal + 타이포 등장
- Stitch Line Draw(섹션 디바이더)
- Archive Hover Texture
- Read More Accent
- Parallax/Scroll Snap(선택)

## 8. 디자인 방향
- 톤&무드: 인디고/헤리티지/로우 텍스처
- 컬러: 인디고 베이스 + 레드 탭 포인트
- 타이포: Oswald(Display) + IBM Plex Sans(Body)
- 레이아웃: 에디토리얼 그리드 + 큰 이미지/짧은 카피

## 9. 기술 스택
- HTML5 / CSS3 / Vanilla JS

## 10. 접근성/반응형/최적화
- prefers-reduced-motion 대응
- 모바일에서 과한 모션 비활성 권장
- data-bg 기반 지연 배경 로딩 및 주요 히어로 이미지 preload 적용
- 모바일 메뉴 aria-expanded/ESC 닫기/포커스 순환 보강
- 모바일 타임라인 overflow QA 및 줄바꿈 안정화

## 11. 해결 과제/차별점
- 커머셜 UI 대신 브랜드 서사 중심 정보 구조
- 텍스처/스티치 요소로 물성 강조
- 서브 페이지 배경 이미지를 지연 로딩해 초기 렌더 부담 완화
- 타임라인 활성 상태를 스크롤 위치와 연동해 서사 흐름 강화

## 12. 결과/배운 점
- 브랜드 서사 중심 정보 구조와 멀티페이지 흐름을 정리
- 인디고 톤/텍스처/스티치 요소를 일관된 비주얼 시스템으로 구현
- 최소 모션으로 몰입감을 유지하는 인터랙션 설계 경험

## 13. 향후 개선
- WebP/srcset 기반 이미지 최적화
- 접근성 심화 점검(alt, aria-label, 키보드 탐색)
- 추가 브라우저/디바이스 반응형 QA
- 코드 정리
